Financial AI Analyzer

Complete user manual for tax document management and bookkeeping

Overview

Financial AI Analyzer is a self-hosted bookkeeping assistant that combines document management with AI-powered analysis. It connects to your Paperless-ngx document library, imports transactions from Gmail receipts, PayPal, Venmo, and bank CSV files, and lets you query everything through a conversational AI interface.

📄 Document Analysis

Pull documents from Paperless-ngx and let AI extract amounts, vendors, dates, and tax categories automatically.

💲 Transaction Import

Import transactions from Gmail, PayPal, Venmo, and any bank's CSV export in a single click.

🤖 AI Chat

Ask questions like "What did I spend on software in 2024?" and get instant answers from your data.

📈 Reports & Exports

Generate profit & loss summaries, export to CSV, JSON, IIF (QuickBooks), OFX, TXF, or PDF.

🏠 Multi-Entity

Track multiple entities (personal, business, farm) in a single installation with per-entity filtering.

🔒 Self-Hosted

All data stays on your server. No third-party cloud sync or SaaS subscriptions required.

System Requirements

  • Docker / Docker Compose on the host server
  • Paperless-ngx instance (for document analysis features)
  • An Anthropic API key (Claude) or OpenAI API key for AI features
  • A modern web browser — Chrome, Firefox, Safari, or Edge

Logging In

Navigate to the app URL (e.g., https://yourserver.example.com/tax-ai-analyzer). You will be redirected to the login page if you are not already authenticated.

  1. Enter your Username and Password.
  2. Check Remember me if you want to stay logged in across browser sessions.
  3. Click Sign In.
Default credentials The default admin account is username admin with password admin123. Change this immediately after first login via Settings → Users.
Session expiry Sessions expire after 24 hours by default. With "Remember me" enabled, the session persists for 30 days via a secure cookie.

Adding or Editing Entities

  1. Go to Settings tab.
  2. Click Manage Entities to expand the entity editor.
  3. Click + Add Entity, enter a name, slug (no spaces), and type.
  4. Click Save. The new entity appears in the entity selector immediately.

Tax Years

Default tax years are 2022–2025. You can add future years in Settings → Tax Years. All transaction filters and report generators respect the currently selected year.

Connecting Gmail

The Gmail importer scans your inbox for receipts, invoices, 1099s, W-2s, and other financial documents. It downloads attachments and creates transactions from the extracted data.

Gmail OAuth Setup

Gmail requires a one-time OAuth 2.0 setup via Google Cloud Console. Navigate to Import → Gmail and click Gmail Setup to launch the guided assistant.

  1. Go to console.cloud.google.com and create or select a project.
  2. Enable the Gmail API: APIs & Services → Library → search "Gmail API" → Enable.
  3. Go to OAuth consent screen. Select External user type (required for personal @gmail.com accounts).
  4. Fill in an app name and your email address. Add your Gmail address as a Test User.
  5. Go to Credentials → Create Credentials → OAuth client ID.
  6. Select application type: Desktop app (not "Web application").
  7. Download the JSON file. Upload it via Import → Gmail → Gmail Setup → Manual Upload tab.
  8. Click Connect Gmail Account to complete the OAuth flow in your browser.
AI-guided setup The Gmail Setup page has an AI-guided assistant mode. Type what you see on screen and the assistant will give step-by-step instructions tailored to your situation.
Testing mode is fine Your Google app will stay in "Testing" mode permanently. This is completely normal and does not affect functionality as long as you are listed as a test user.
Credential type must be "Desktop app" A common mistake is choosing "Web application" instead. If you receive redirect URI errors during authorization, re-create the credentials as a Desktop app.

Running a Gmail Import

Once credentials and a token are configured, running an import is one click.

  1. Go to Import → Gmail.
  2. Select the Entity and Tax Year to associate with imported transactions.
  3. Click Start Gmail Import.
  4. The job runs in the background. Watch the progress in Import → Import Jobs.

What gets imported

The importer searches for emails matching these keywords: receipt invoice statement payment billing 1099 W-2 tax order. For each matching email, it:

  • Extracts the subject, sender, date, and body.
  • Downloads PDF/image attachments and sends them to Paperless-ngx for OCR and storage.
  • Uses AI to extract vendor name, amount, category, and tax year from the content.
  • Creates a transaction record with the extracted data.
Years covered By default the importer covers 2022–2025. Change this in Settings → Gmail Years, or set the GMAIL_YEARS environment variable.

PayPal & Venmo CSV Import

Export a transaction CSV from your PayPal or Venmo account and upload it directly.

Exporting from PayPal

  1. Log into PayPal and go to Activity.
  2. Click Download (top right), select CSV format.
  3. Choose your date range (full year recommended) and download.

Exporting from Venmo

  1. Log into Venmo and go to Statements (Settings → Privacy → Download your data, or the Statements link).
  2. Select the year and download the CSV.

Uploading

  1. Go to Import → PayPal (or Venmo).
  2. Select the entity and tax year.
  3. Click Choose File and select your CSV.
  4. Click Import. The job runs in the background.
Duplicate handling Transactions are upserted by a unique key (date + amount + description) so re-importing the same CSV will not create duplicates.

Bank CSV Import

Most banks allow exporting a CSV of transactions. The importer supports a generic format with auto-detection for common bank layouts.

Supported formats

  • Chase Bank CSV (Date, Description, Amount columns)
  • Bank of America CSV
  • Wells Fargo CSV
  • Generic OFX-to-CSV exports
  • Any CSV with date, description (or memo), and amount columns

Steps

  1. Download your bank statement as CSV from your bank's website.
  2. Go to Import → Bank CSV.
  3. Select entity and tax year.
  4. Upload the CSV and click Import.
Column mapping If the importer cannot detect your bank's format, check the CSV Format Reference section below for the expected column names.

URL Importer

The URL importer downloads a CSV or JSON file from a URL on a recurring schedule — useful for integrations that expose a downloadable export endpoint.

  1. Go to Import → URL.
  2. Enter the URL, file format (CSV/JSON), entity, and tax year.
  3. Set the polling interval (in minutes).
  4. Click Add URL Poller. The background scheduler will fetch the URL at the specified interval.

Authentication

If the URL requires HTTP Basic authentication, enter the credentials in the Username and Password fields. For Bearer tokens, enter the token in the Password field and leave Username empty.

Cloud Storage Import

The cloud adapter connects to Google Drive, Dropbox, or Amazon S3 to scan for CSV and PDF files and import them automatically.

Supported Providers

ProviderCredential required

Configuration

  1. Go to Import → Cloud.
  2. Select a provider and enter the required credentials.
  3. Specify the folder or bucket path to scan.
  4. Click Connect & Scan. A background job will fetch matching files.
Optional feature Cloud adapters require optional Python packages (google-api-python-client, dropbox, boto3). If not installed, the Cloud tab will display an installation notice.

Transactions

The Transactions tab shows all financial records — income and expenses — across all import sources.

Adding Transactions Manually

  1. In the Transactions tab, click + Add Transaction.
  2. Fill in: Date, Description, Amount (positive for income, negative for expense), Category, Entity, Tax Year.
  3. Optionally add a Vendor, Notes, and Source tag.
  4. Click Save.

Filtering & Searching Transactions

The filter bar at the top of the Transactions tab supports:

  • Search — full-text search across description, vendor, and notes fields.
  • Category — filter to a single category.
  • Type — Income, Expense, or All.
  • Date range — start and end date pickers.
  • Entity / Year — inherited from the global selectors in the top bar.

Click Apply Filters or press Enter to update the list. Click Reset to clear all filters.

Categorizing Transactions

Each transaction has a category used in reports and tax exports. Categories are based on IRS Schedule C / Schedule F lines for business and farm entities, and common personal finance buckets for personal entities.

Changing a category

  1. Click anywhere on a transaction row to expand it.
  2. Click the Category field and select a new category from the dropdown.
  3. Changes are saved automatically.

AI Re-categorize

To let AI suggest a category for a transaction, click the AI button next to the category field. The AI will analyze the description and vendor name and suggest the most appropriate category.

AI Document Analysis

The AI analyzer polls Paperless-ngx for new documents and automatically extracts financial data. You can also trigger analysis manually.

Triggering analysis manually

  1. Go to the Documents tab.
  2. Click Run AI Analysis (top right).
  3. The analyzer will process any unanalyzed documents. Progress is shown in the activity feed.

Vision AI fallback

When the analyzer detects poor OCR quality (e.g., scanned images, handwritten forms), it automatically falls back to sending document images directly to Claude's Vision API. Documents processed this way are tagged aianomaly:vision_ai_extracted in Paperless-ngx.

Paperless-ngx connection AI analysis requires a running Paperless-ngx instance. Configure the URL and API token in Settings → Paperless.

AI Chat Assistant

The Chat tab provides a conversational interface to your financial data. Ask questions in plain English and receive answers drawn from your transactions, documents, and summaries.

Starting a new chat

  1. Go to the Chat tab.
  2. Click New Session.
  3. Type your question in the input box and press Enter or click Send.

Example questions

  • "What are my total business expenses for 2024?"
  • "Which vendor did I pay the most in 2023?"
  • "List all income transactions over $500 for the VoIPGuru entity."
  • "What categories do I still need receipts for?"
  • "Summarize my Schedule C deductions for 2024."
  • "Are there any transactions that look like duplicates?"

Chat history

All sessions are saved. Click any session in the left panel of the Chat tab to reload the conversation. Sessions can be deleted by hovering the session title and clicking the trash icon.

Context matters The AI uses your current entity and year filters when answering. Switch entities using the top bar before asking entity-specific questions.

Reports & Exports

The Reports tab generates financial summaries and downloadable export files.

Available reports

  • Profit & Loss Summary — income minus expenses by category for the selected entity and year.
  • Category Breakdown — bar chart and table showing spending by category.
  • Monthly Trend — income and expense totals by month.
  • Top Vendors — highest-spend vendors for the selected period.

Generating an export

  1. Select the entity and tax year using the top bar.
  2. In the Reports tab, choose the export format.
  3. Click Generate. The file is prepared in the background.
  4. Once ready, click Download.

User Management

The Users tab is visible to admin accounts only. It allows creating, editing, and deactivating user accounts.

User roles

RoleAccess level
adminFull access including Users tab and all settings
standardFull access except Users tab and destructive settings

Adding a user

  1. Go to Users tab.
  2. Click + Add User.
  3. Enter username, email, password, and role.
  4. Click Create User.

Resetting a password

  1. Find the user in the Users list.
  2. Click Reset Password in the action menu.
  3. Enter a new password and confirm.
  4. Click Set Password.

Deactivating a user

Click the Deactivate button next to a user. Deactivated users cannot log in but their data is retained. You can reactivate them later.

Admin protection You cannot deactivate your own account or the last remaining admin account.

Troubleshooting

Login issues

ProblemSolution
"Invalid username or password"Check username spelling. Default credentials: admin / admin123.
Redirected to login on every pageClear browser cookies. Ensure the server is accessible via HTTPS (required for secure cookies in production).
Account locked / inactiveAn admin must reactivate the account in the Users tab.

Import issues

ProblemSolution
Gmail import fails immediatelyCheck that credentials.json and the OAuth token are both present in Settings → Gmail status. Re-run Gmail Setup if the token is expired.
CSV import shows 0 recordsCheck that the file has the expected column names. See CSV Column Reference above. Try opening the file in a text editor to inspect the headers.
Import job stuck at "running"Check the container logs: docker logs tax-ai-analyzer. Restart the container if the job has not progressed in 10+ minutes.

AI / LLM issues

ProblemSolution
Chat returns "API key not configured"Go to Settings and enter your Anthropic or OpenAI API key. Click Test LLM to verify.
AI analysis is slowLarge documents take longer. Check container resource limits in docker-compose.yml. Reduce parallel analysis workers if memory is constrained.
AI gives wrong categoriesYou can manually override categories on any transaction. The AI uses description and vendor — adding more context in the Notes field improves future suggestions.

Paperless-ngx connection

ProblemSolution
Documents tab shows 0 documentsVerify Paperless URL and API token in Settings. Click Test Paperless. Ensure Paperless is running: docker ps | grep paperless.
Documents show "error" badgeCheck container logs for the specific document ID. Common cause: OCR extracted no text and Vision AI fallback is not enabled.

Checking logs

# View real-time logs for the analyzer
docker logs -f tax-ai-analyzer

# View last 100 lines
docker logs --tail 100 tax-ai-analyzer

# View Paperless-ngx logs
docker logs -f tax-paperless-web
Activity feed The Dashboard tab shows a real-time activity feed of recent operations. This is often the fastest way to see what happened during an import or analysis run without needing to check logs.
