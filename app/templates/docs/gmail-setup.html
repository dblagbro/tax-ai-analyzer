Gmail OAuth Setup

Gmail requires a one-time OAuth 2.0 setup via Google Cloud Console. Navigate to Import → Gmail and click Gmail Setup to launch the guided assistant.

  1. Go to console.cloud.google.com and create or select a project.
  2. Enable the Gmail API: APIs & Services → Library → search "Gmail API" → Enable.
  3. Go to OAuth consent screen. Select External user type (required for personal @gmail.com accounts).
  4. Fill in an app name and your email address. Add your Gmail address as a Test User.
  5. Go to Credentials → Create Credentials → OAuth client ID.
  6. Select application type: Desktop app (not "Web application").
  7. Download the JSON file. Upload it via Import → Gmail → Gmail Setup → Manual Upload tab.
  8. Click Connect Gmail Account to complete the OAuth flow in your browser.
AI-guided setup The Gmail Setup page has an AI-guided assistant mode. Type what you see on screen and the assistant will give step-by-step instructions tailored to your situation.
Testing mode is fine Your Google app will stay in "Testing" mode permanently. This is completely normal and does not affect functionality as long as you are listed as a test user.
Credential type must be "Desktop app" A common mistake is choosing "Web application" instead. If you receive redirect URI errors during authorization, re-create the credentials as a Desktop app.
